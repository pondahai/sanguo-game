# 三國志網頁版 — 設計計畫

> 狀態：討論中草案（2026-07-17）
> 姊妹專案：[sanguo-wiki](https://github.com/pondahai/sanguo-wiki)（本地 `Downloads\sanguo`）

## 一、遊戲定位與範圍

**範本**：光榮三國志 I 的核心循環（內政養國力 → 徵兵 → 出征 → 吞併），加上三國志 II 的少量改良（火計、外交選項、單挑）。

**不做**：即時制、複雜兵種相剋、結婚生子、艦隊——控制範圍是網頁版能完成的關鍵。

**勝利條件**：統一全部州郡。

**劇本**：先做「189 年董卓亂政」開局；架構上支援多劇本（劇本＝一份 JSON：年份＋各州歸屬＋武將配置）。

**規模**：
- 地圖：約 40 個「州郡」節點（三國志 I 是 58 國，網頁版縮減較好排版）
- 武將：sanguo-wiki 的 137 人全收；主要 30～40 人手工設定數值與臉譜，其餘規則生成
- 一場遊戲時長目標：2～4 小時

## 二、核心系統設計

### 1. 回合結構（月制）

每月一回合，玩家依序對每個自己的州郡下**一道指令**（三國志 I 的經典限制，策略深度來源——永遠做不完想做的事）：

```
玩家回合：逐州下令 → AI 各勢力行動 → 月結算（收成/俸祿/事件）→ 下月
```

- 每年 7 月收糧、1 月收金（稅率可調，太高掉民忠）
- 隨機事件：豐收、蝗災、瘟疫、武將病死（史實年份到了觸發史實死亡）

### 2. 內政指令

| 指令 | 效果 | 主要依賴數值 |
|---|---|---|
| 開墾 | 土地開發度↑ → 糧收入↑ | 政治 |
| 治水 | 防洪水災、開發度上限↑ | 政治 |
| 商業 | 商業值↑ → 金收入↑ | 智力+政治 |
| 施捨 | 民忠↑（花糧） | 魅力 |
| 徵兵 | 兵力↑（花金糧、民忠↓） | 魅力 |
| 訓練 | 訓練度↑ | 武力 |
| 搜索 | 找在野武將、登用 | 魅力 |

### 3. 軍事與戰鬥

出征採**簡化戰棋**：獨立戰鬥畫面，方格地圖，最多各帶 5 支部隊（每部隊＝一名武將＋兵力）。回合制移動＋攻擊，30 天內攻不下城視為撤退。

保留四要素：
1. **地形**（平地/山/河/城）
2. **火計**（智力判定）
3. **攻城**
4. **單挑**（相鄰敵將可發起，武力+隨機數；呈現用文字播報＋臉譜對峙畫面）

兵糧耗盡部隊潰散——糧道是早期三國志的靈魂，保留。

### 4. 人事與外交

- 武將有**忠誠**與**義理**隱藏值：忠誠低會被挖角/叛變；俘虜敵將可招降（義理高如關羽極難降）
- 外交：同盟、停戰、要求共同出兵、勸降（送金＋使者魅力判定）
- 君主死亡→繼承人接位；無繼承人→勢力瓦解武將四散

### 5. 武將數值

早期五維：**武力、智力、政治、魅力**＋隱藏**義理/野心**。同一組數值同時驅動臉譜生成，一份資料兩用。

## 三、資料模型

```
data/
  scenario_189.json    劇本：勢力、州郡歸屬、武將初始配置
  provinces.json       州郡：id、名稱、鄰接表、地圖座標、初始開發度
  officers.json        武將：由 sanguo-wiki characters.py 轉出
```

`officers.json` 每筆：

```json
{
  "id": "guanyu",
  "name": "關羽", "zi": "雲長",
  "birth": 160, "death": 219,
  "stats": { "wu": 97, "zhi": 75, "zheng": 62, "mei": 93, "yi": 100 },
  "face": { "color": "red", "beard": 2, "hat": "scarf", "fab": ["#3E7A50", "#28543A"] },
  "wiki": "characters/guanyu.html"
}
```

- `face` 缺省時由 `id` hash 決定性生成——137 人不必逐一手調
- `wiki` 連回 sanguo-wiki 人物頁：遊戲中點武將卡→看列傳（差異化特色）
- 生卒年驅動史實死亡事件；sanguo-wiki 的 facts 清單可再轉成「史實事件表」

## 四、技術架構

**原則**：無框架、無 build step、純靜態——與 sanguo-wiki 靜態站同一哲學，開 `index.html` 就能玩，可直接上 GitHub Pages。

```
sanguo-game/
  index.html
  css/main.css
  js/
    main.js          啟動、劇本載入
    state.js         遊戲狀態單一物件 + 存讀檔(localStorage/JSON下載)
    face.js          臉譜 SVG 生成器（獨立模組，可回饋給 wiki）
    map.js           州郡地圖渲染（SVG：節點+鄰接線）
    turn.js          回合流程、月結算、事件
    commands.js      內政/人事/外交指令實作
    battle.js        戰棋畫面（獨立 state machine）
    ai.js            電腦勢力決策
    ui.js            面板、對話框、訊息日誌
  data/ (如上)
  tools/
    export_officers.py   從 sanguo-wiki characters.py 轉出 officers.json
```

技術要點：
- **地圖用 SVG 節點圖**而非擬真地圖：40 州郡＝節點＋連線，勢力上色。省最花時間的地圖美術，且與臉譜色塊美學一致；日後換手繪底圖不動介面程式
- **戰鬥畫面用 CSS Grid** 方格＋小尺寸臉譜當部隊棋子
- **存檔**：state 整包 JSON 進 localStorage，另供下載/上傳
- **AI**：規則式——弱國先內政、兵糧優勢 >1.5 且鄰接才出征、優先打最弱鄰居、嘗試挖角低忠誠敵將

## 五、臉譜模組規格（face.js）

> 2026-07-17 定案：**八位元點陣風格**。生成器在低解析度格子上作畫（調色盤索引陣列），
> 最後渲染成 SVG `<rect>`（同色橫向連續格合併）＋ `shape-rendering="crispEdges"`。
> 向量輸出、點陣外觀：任意縮放銳利、換色即換字串、一張臉約 150~200 rect（4~5 KB）。

### 雙解析度策略

| 版本 | 畫布 | 用途 |
|---|---|---|
| 細版 | **32×46**（臉佔 32×40，頭頂留 6 列給冠帽） | 武將卡、單挑畫面、列傳 |
| 粗版 | 20×26 | 戰場棋子（32px 小尺寸），自動退化 |

兩套生成器共用同一組參數；粗版只保留底色＋眉＋鬚（遠看粗像素、近看細像素）。

### API

```js
renderFace(params, size)  → SVG string   // size 決定用細版或粗版
faceFromId(officerId)     → params        // 確定性 hash，供未手調武將
```

### 參數

```js
{
  color: 'red',        // 底色 7 選 1（性格語義，見下）
  wu: 97, zhi: 75,     // 連續值：驅動眉形斜度/粗細、眼型（wu≥90 環眼）
  beard: 2,            // 鬍鬚：0 無鬚 / 1 短髭 / 2 長髯 / 3 虯髯
  hat: 'scarf',        // 頭冠：crown 冕 / helmet 兜鍪 / scarf 綸巾 / civil 進賢冠 / topknot 束髮髻
  fab: ['#3E7A50','#28543A'],  // 綸巾布色（選填；關羽綠巾）
  cheek: 'butterfly',  // 臉頰紋（選填；張飛蝴蝶紋，取代額頭智力紋）
  old: true            // 老將旗標：鬍鬚+眉毛轉霜白（可接年齡系統：60 歲自動開）
}
```

**底色語義**：紅忠義 / 黑剛直 / 白多謀 / 黃梟勇 / 藍驍悍 / 紫沉穩 / 粉老成。
每色一組四階（skin / shade / dark / light），臉緣自動描 shade 一圈做立體感。

**固定調色盤**（不隨臉色變）：墨黑+霜白（眼、鬚）、鬚紋理雙墨色、金屬灰雙階（盔）、
金雙階（冠）、布色雙階（巾，可換）、紅纓紅、白鬚雙階。全部臉共用一張 17 色索引表。

**三國志II風 16 色母盤＋抖色**（2026-07-17 加入）：所有顏色收斂到 `K16` 十六色母調色盤
（PC-98/KOEI 色感），中間調用 2×2 棋盤格抖色（`(x+y)` 奇偶交錯雙色）——紫臉＝藍紅抖色、
陰影＝本色+暗色抖色、白鬚灰白交錯。資料端自訂色（如關羽綠巾）經 `Face.quant()` 量化到母盤。
戰場地形（綠野/褐山/藍河/金城）與火場用 `Face.ditherCSS()` 的 CSS 棋盤格；
地圖勢力色渲染時量化；UI 主題＝暖黑底/焦茶框/山吹金。

### 部件庫（蓋章式，畫在臉圖層之上）

| 部件 | 樣式 | 自動指定規則（可手動覆蓋） |
|---|---|---|
| 冕冠 crown | 金平天板+兩側垂珠 | 現任君主（滅國淪為部下自動摘冠） |
| 兜鍪 helmet | 金屬圓盔+金脊+紅纓 | 武力 ≥80 |
| 綸巾 scarf | 布巾+摺紋+垂尾，布色可換 | 軍師職 |
| 進賢冠 civil | 黑高冠前低後高+金冠帶 | 政治 ≥70 |
| 束髮髻 topknot | 頂髻+金束帶 | 其他/在野（預設） |
| 額紋 | 智力菱紋（zhi≥60） | 自動 |
| 蝴蝶紋 butterfly | 眉外蝶翼+額心蝶身+眼窩翼脈 | 手工指定（張飛等） |
| 老將白鬚 old | 鬚+眉轉霜白雙階 | 年齡 ≥60 自動 |

### 手調名單原則

主要 30~40 人手工鎖定招牌造型（關羽紅臉綠巾長髯、張飛黑臉蝴蝶紋虯髯、
黃忠白鬚兜鍪…），其餘由 `faceFromId()` hash 生成。理論組合
7 色 × 4 鬚 × 5 冠 × 2 紋 × 老/壯 × 眉眼連續值 ≫ 137，不撞臉。

### 延伸玩法（實作成本低，因為是格子資料）

- 單挑敗北：隨機挖格 / 整臉錯位一格的碎裂特效
- 勢力變色：戰場棋子外框上勢力主色
- 年齡敘事：同一武將隨遊戲年份從壯年臉打到白眉白鬚

## 七、待討論的取捨

1. **州郡節點圖 vs 擬真地圖**——節點圖最省工且風格統一，但少「看地圖」沉浸感
2. **每州每月一指令**（三國志 I 限制）——精髓 vs 節奏慢
3. **方格戰棋 vs 六角格**
