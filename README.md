# 三國志（網頁版）

以光榮《三國志 I/II》早期玩法為範本的純靜態網頁策略遊戲。
零框架、零 build step、零圖片素材——所有武將頭像都是程式生成的
**國劇臉譜式八位元點陣 SVG**，配三國志 II 風 16 色調色盤與棋盤格抖色。

**▶ 立即遊玩**：<https://pondahai.github.io/sanguo-game/>
　**臉譜工房**（自己捏一張臉譜）：<https://pondahai.github.io/sanguo-game/face.html>

## 特色

- **165 名武將**，資料轉自姊妹專案 [sanguo-wiki](https://github.com/pondahai/sanguo-wiki)
  （《三國演義》全文事實抽取）——配角數值由原著出場統計推導
- **程序生成臉譜**：底色 7 色（國劇性格語義：紅忠義/白多謀/黑剛直…）×
  鬍鬚 4 式 × 頭冠 5 式（冕/兜鍪/綸巾/進賢冠/束髮髻）× 蝴蝶紋 × 老將白鬚，
  眉眼由武力智力數值驅動；紫臉是藍紅抖色抖出來的（PC-98 正宗手法）
- **點頭像看列傳**：浮動視窗直連 sanguo-wiki 的人物生平頁
- **演義生成**：填入你自己的 LLM 端點（OpenAI 相容），根據存檔的「戰局實錄」
  寫一部**你的世界線**的章回小說——只准根據實錄寫作、嚴禁模型憑記憶摻入
  真實演義情節（防幻覺鐵則承襲 sanguo-wiki）；年表濾掉例行結算、明定時序鐵則，
  防長文本「迷失在中間」的時間錯亂。設定只存瀏覽器，金鑰只送你填的端點
- **三國志 I 式玩法**：40 州郡節點圖、月回合、每州每月一道指令、
  方格戰棋（地形/火計/糧道）、單挑臉譜對峙、挖角/同盟/勸降、史實病逝與繼承
- **189 年董卓亂政**劇本，14 勢力可選

## 本地執行

```
python -m http.server 8377
# 開 http://localhost:8377
```

## 資料管線

```
sanguo-wiki (characters.py + facts/)
  → tools/export_officers.py
  → data/officers.json / officers.js
```

主要 67 人手工鎖定數值與招牌造型（關羽紅臉綠巾長髯、張飛蝴蝶紋虯髯…），
其餘 98 人由事實句數（活躍度）＋確定性 hash 推導。詳細設計見 [DESIGN.md](DESIGN.md)。

## 開發緣起

本專案由作者與 Claude（Claude Code）在對話中共同完成。追根究柢，一切始於
[sanguo-wiki](https://github.com/pondahai/sanguo-wiki) 專案的第一句提示詞：

> 「如果我拿一本三國演義原文文字檔 能夠一鍵變成wiki資料庫嗎」

那句話催生了 137 位人物、每條事蹟附回數出處的三國演義 wiki 資料庫。
資料庫建成之後，才有了本專案的開場白：

> 關於三國志電腦遊戲 我想要實現在網頁程式 可能以早期玩法為範本
> 然後參考 我們之前一起整理的三國演義 https://github.com/pondahai/sanguo-wiki
> 其中比較難辦的是人物臉譜 可能要用類似國劇臉譜的方式以色塊組合 你覺得呢

於是 wiki 的人物字典成了武將名冊、逐回事實句數成了配角數值、
人物生平頁成了遊戲內的列傳浮窗——一本小說原文，先變成資料庫，再變成遊戲。

### 開發歷程簡述

1. **臉譜風格探索**——先做平滑 SVG 色塊臉譜的概念驗證，
   再依「想要八位元時代感」的方向改成點陣式：格子陣列作畫、SVG rect 渲染。
   比較 20×26 與 32×40 兩種解析度後定案雙軌制（細版看卡片、粗版當棋子），
   陸續加入頭冠部件（冕/兜鍪/綸巾/進賢冠/束髮髻）、張飛蝴蝶紋、老將白鬚
2. **設計定案**——寫成 DESIGN.md：三國志 I 式玩法（月回合、每州一指令、
   方格戰棋）、40 州郡砍併方案、資料模型與五個里程碑
3. **M1 資料管線**——`export_officers.py` 從 sanguo-wiki 轉出 165 名武將；
   配角數值由原著出場統計推導。40 州郡鄰接圖、189 年劇本、武將名冊頁
4. **M2 內政循環**——地圖、月回合、七道內政指令、稅收秋收、事件、
   史實病逝與繼承、存讀檔。跑三年模擬調平衡（初版全地圖會餓死民變）
5. **M3 戰棋**——出征/移動、13×9 戰場（地形 hash 生成）、火計、
   單挑臉譜對峙動畫、兵糧、攻城佔領、俘虜招降（依義理判定）
6. **M4 AI 與外交**——AI 擴張與互咬、玩家守城戰、自動戰結算、
   賞賜/挖角/同盟/勸降
7. **M5 打磨發佈**——結局畫面、AI 節奏、GitHub Pages 上線
8. **復古美術**——全面改用三國志 II 風 16 色母調色盤＋棋盤格抖色
   （紫臉＝藍紅抖色，PC-98 正宗手法），UI 換 KOEI 暖黑金
9. **wiki 整合**——點武將頭像開浮動列傳視窗，直連 sanguo-wiki 人物頁
10. **演義生成器**——把 sanguo-wiki 的防幻覺哲學反過來用：wiki 是
    「原著→事實清單→生平」，這裡是「**存檔→戰局實錄→一部新的演義**」。
    遊戲把每件大事寫進實錄（攻陷、單挑、被俘、病逝、繼位、覆滅…），
    玩家填自己的 LLM 端點（自動抓模型清單），由 LLM 依鐵則寫章回小說：
    只准根據實錄、嚴禁摻入真實演義的記憶情節。
    以 DGX Spark 上的 Qwen3.6-35B 實測調校：關閉 thinking 模式才能串流正文；
    第一輪生成漏出「十八路諸侯共推盟主」（實錄根本沒有會盟），
    加鐵則明文禁止後消除。最有趣的插曲：測試時看到「曹叡效力孫堅、
    諸葛瑾出仕馬騰」，以為是模型幻覺，查了存檔才發現——那是遊戲裡
    AI 勢力用「搜索」指令真實登用的在野武將。**模型沒有腦補，
    是那條世界線真的長這樣**。這正是整個功能想要的：荒謬感全部
    來自遊戲本身，一分都不是 LLM 編的
11. **演義防時間錯亂**——長期存檔實測發現 LLM 會把「終局現況」當
    「開局設定」（劉備統治 35 年，模型卻讓繼位的姜維從頭當主公）：
    數百條收稅、秋收流水帳沖淡了主公更迭的權重，典型的
    「迷失在中間」注意力缺陷。對策：年表剔除例行結算（玩家內政令保留）、
    明示【天下大勢】是結尾快照非開局、新增時序鐵則（繼位只能發生在
    記載年月）。閱讀器同時補上 markdown 渲染
12. **體驗打磨**——戰鬥可走格提示加強、武將過世訃聞彈窗、
    出征選將高亮/未選黯淡、兵糧改拉桿、戰場內可捲動事件紀錄欄
    （陣中隨時回查整場經過）

過程中所有機制（經濟平衡、AI 決策、單挑公式）都以瀏覽器自動化實測驗證後才定案。
